Introdução

O Simple Design System (SDS) fornece um conjunto de recursos para auxiliar e agilizar o design e desenvolvimento dos produtos e serviços da UNGP, a partir de princípios, diretrizes de design e desenvolvimento, com exemplos e melhores práticas.

Alguns dos objetivos incluem:

Linguagem visual e escrita consistente e apropriada Boas práticas de acessibilidade Foco na usabilidade dos sistemas Código limpo e bem estruturado

Instruções para Designers:

Ler e aprender os princípios e diretrizes do SDS, utilizando como recurso para consulta sempre que necessário. Se familiarizar com os componentes de UI para utilizar em seus projetos. Ir para a página de Downloads para começar a utilizar os arquivos de design.

Instruções para Desenvolvedores

Ler e aprender sobre as diretrizes e convenções de Marcação e Estilo adotadas no desenvolvimento do SDS. Se familiarizar com os componentes de UI e sua documentação seguindo os exemplos de marcação semanticamente corretas. Conhecer e contribuir nos projetos paralelos que utilizam o SDS como base. Ir para a página do Projeto no GIT para começar a utilizar os arquivos de design.
